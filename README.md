# MusicBar

MusicBar 是一个贴靠 Windows 任务栏的开源音乐快捷播控工具。它优先通过 Windows 全局媒体会话识别当前播放器，并为部分播放器提供本地适配能力。

## 第一版功能

- 自动选择 Windows 当前活跃的媒体会话
- 显示歌曲名、歌手、播放器来源和封面
- 播放、暂停、上一首、下一首
- 在播放器开放能力时切换列表循环、单曲循环和关闭循环
- 贴靠主屏幕任务栏工作区，可拖动，也可在右键菜单中重新贴靠
- 自动跟随任务栏的明暗主题、透明效果和 Windows 强调色
- 自动读取任务栏实际高度，并以内嵌式紧凑布局覆盖任务栏左侧空白区域
- 单实例运行：重复双击不会创建多个 MusicBar 进程
- 切换到其他应用后自动维护窗口层级，避免被任务栏遮挡且不抢输入焦点
- 加载本地 `.lrc` 歌词，并按当前播放时间同步显示
- 网易云音乐自动歌词：从本地播放列表精确匹配歌曲 ID，再读取对应的同步歌词
- 网易云音乐后备识别：即使先打开网易云、后打开 MusicBar，也能从播放器窗口识别当前歌名和歌手
- 通过 MusicBar 启动网易云时读取其本机界面的原生进度，提高歌词时间轴精度
- 汽水音乐原生歌词：只读解析本地播放缓存，按歌曲名、歌手和时长精确匹配 KRC 同步歌词
- 对不持续上报播放位置的播放器使用本地时间轴锚点，保证歌词能够逐行推进
- 启动时歌曲已经播放到中途，可右键输入播放器当前的 `分:秒` 校准；也可前后微调 1 秒
- LRCLIB 在线歌词后备源，按歌名、歌手、专辑和时长进行置信度匹配
- LRCLIB 成功查询后自动保存到用户本地缓存，再次播放同一首歌时无需联网
- 歌词提供者自动降级：手动 LRC → 播放器专用提供者 → 在线歌词
- 播放器不支持的按钮会自动禁用

收藏按钮在第一版中有意禁用，因为 Windows 没有跨播放器的统一收藏命令，后续需要为不同播放器接入账户授权或专用适配器。

自动歌词会把当前歌曲的公开元数据发送给对应在线歌词服务。LRCLIB 查询成功的歌词保存在 `%LOCALAPPDATA%\MusicBar\lyrics-cache\v1`。网易云适配器只读取本机播放列表文件、窗口标题和本机播放器页面状态；汽水音乐适配器只读本地歌词缓存。它们都不读取登录 Cookie、不修改播放器数据，也不注入播放器进程。网易云原生进度通道只监听 `127.0.0.1`，不会向局域网开放调试端口。

## 运行

要求 Windows 10 1809 或更高版本，开发环境需要 .NET 9 SDK。

```powershell
dotnet run --project .\MusicBar\MusicBar.csproj
```

也可以直接运行已经发布的版本：

```text
artifacts\publish\MusicBar.exe
```

## 安装包

安装版默认安装到当前用户的 `%LOCALAPPDATA%\Programs\MusicBar`，不需要管理员权限，包含开始菜单快捷方式、可选桌面快捷方式和标准卸载入口。

安装包输出位置：

```text
artifacts\installer\MusicBar-Setup-0.2.0-alpha.13-win-x64.exe
```

安装 Inno Setup 6 后，可以重新构建安装包：

```powershell
.\installer\build-installer.ps1
```

在控件空白处拖动可改变位置。右键打开菜单，可以加载当前歌曲对应的 LRC 文件、重新贴靠任务栏或退出。

## 验证

```powershell
dotnet test .\MusicBar.sln
dotnet build .\MusicBar.sln -c Release
```

## 当前边界

- 向 Windows 注册系统媒体会话的播放器可以获得最完整的识别和控制能力；网易云音乐另有窗口后备识别。
- 网易云音乐、汽水音乐和 LRCLIB 已支持自动歌词；汽水音乐仅能读取已经由播放器缓存到本机的歌曲歌词。
- 网易云在 MusicBar 之前以普通方式启动时可以识别歌曲，但精确时间轴能力不及从 MusicBar 启动网易云。
- 收藏功能需要播放器专用适配器。
- 第一版跟随主任务栏；多显示器副任务栏跟随和任务栏自动隐藏联动将在后续版本加入。
